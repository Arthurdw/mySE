Install guide:

To install the LTS mySE module you can use PIP! 

            >>> pip install mySE

A basic example:

Example code, this code creates and/or fetches the token from the API.
Then it creates a Client Object and retrieves the id. 

            
                from mySE import mySE

                local_url, mail, token = "http://127.0.0.1:5000/", "mail@mail.mail", None

                try:
                    token = mySE.gen_token(local_url, mail)
                except mySE.error.UnauthorizedError:
                    token = mySE.get_token(local_url, mail)

                client = mySE.Client(local_url, token)
                print(f"Client ID: {client.id}")

API reference:

mySE.get_token(email)

Retrieves a valid Client token using a email.

Parameter:

  • email (Required: string) - Your email address!

Raises:

  • UnauthorizedError – No token exists with this email, use gen_token instead.

mySE.gen_token(email)

Generates a valid Client token using a email.

Parameter:

  • email (Required: string) - Your email address!

Raises:

  • UnauthorizedError – A token already exists using this email, use get_token instead.

mySE.mySE.Client(url, token)

Represents a mySE Client, this is the main object for this API wrapper.
This is object is used to interact with the mySE API.
Type: Function
Returns: mySE Client Object

Parameters:

  • url (Required: string) - The mySE server link. Eg. use "http://127.0.0.1:5000/" if you're running the server locally on port 5000.
  • token (Required: string) - Your unique mySE token. If you don't have a token yet generate it using gen_token.

Raises:

  • UnauthorizedError – The wrong credentials are passed.

Client.id

The current Client its user ID.
Type: Property
Returns: integer

Client.logs

Fetches all Client linked log objects.
Type: Property
Returns: mySE Log object

Client.add_log(light)

Adds a log under the current Client its id.
This also executes/returns a Client.logs.
Type: Function
Returns: mySE Log object

Parameter:

  • light (Required: boolean) - If the light was on/off in the room!

logs.Log

The base log object used in logs & add_log.

Log.id

The selected Log its unique ID.
Type: Property
Returns: integer

Log.light

Was the light on when this Log got recorded.
Type: Property
Returns: boolean

Log.time

The time when this Log got recorded. (It can be inaccurate from around 4-200s)
Type: Property
Returns: DateTime object

error.UnauthorizedError

Exception that occurs when an invalid token/email has been given.
