API documentation

Authorization

The API requires a Authorization in the header with a unique token.
Having no authorization token will result in a 401 Unauthorized error response!

/logs/

Retrieves all logs linked to your auth token.

GET request:

Fetches all linked log objects.
Requires an Authorization token in the header.
Value Type Description
id integer The auth token its linked client id.
statusCode integer The status code for the request. Eg: 201, 401
logs object

logs object

Logs retrieved trough a logs request.

Value Type Description
light boolean Indicates if the light in the room was on while the log got recorded.
id integer The unique ID for each log.
time object

time object

A basic time object.

Value Type Description
year int The year of the log.
month int The month of the log.
day int The day of the log.
hour int The hour of the log.
minute int The minute of the log.
second int The second of the log.
microsecond int The microsecond of the log.

POST request:

Adds a log object to the DB.
Requires an Authorization token in the header.

Input:

Value Type Description
light boolean If the light was on or off in the room.

Returns:

Value Type Description
id integer The auth token its linked client id.
statusCode integer The status code for the request. Eg: 201, 401
logs object

/token/

Create and retrieve your auth token with your email.

GET request:

Fetches the auth code that is linked with the given mail.
The get request requires a mail attribute with a valid email.
Value Type Description
id integer The auth token its linked client id.
token string The unique auth code!
statusCode integer The status code for the request. Eg: 201, 401

POST request:

Creates a unique auth token!
The POST request requires a mail attribute with a valid email.

Input:

Value Type Description
mail string Your email!

Returned value:

Value Type Description
token string The unique auth code!
statusCode integer The status code for the request. Eg: 201, 401
